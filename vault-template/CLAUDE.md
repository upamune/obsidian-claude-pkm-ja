## 利用可能なスキル

スキルは`/skill-name`で呼び出すか、関連する場合にClaudeが自動的に実行します。

| スキル | 呼び出し | 目的 |
|-------|------------|---------|
| `daily` | `/daily` | デイリーノートの作成、朝/昼/夕方のルーチン |
| `weekly` | `/weekly` | 週次レビューの実行、振り返りと計画 |
| `push` | `/push` | 変更をGitにコミット＆プッシュ |
| `onboard` | `/onboard` | Vaultの完全なコンテキストを読み込み |
| `goal-tracking` | (自動) | 目標カスケード全体の進捗を追跡 |
| `obsidian-vault-ops` | (自動) | Vaultファイルの読み書き、wikiリンクの管理 |

### 進捗の可視化

スキルとエージェントはセッションタスクツールを使用して、複数ステップの操作中に進捗を表示します：

```
[スピナー] デイリーノートを作成中...
[スピナー] 未完了タスクをプル中...
[完了] 朝のルーチン完了 (4/4 タスク)
```

セッションタスクは一時的な進捗インジケーターです。実際のタスク項目はデイリーノート内のマークダウンチェックボックスとして残ります。

## デイリーワークフロー

### 朝（5分）
1. `/daily`を実行して今日のノートを作成
2. 1つの主要フォーカスを特定
3. 昨日の未完了タスクをレビュー
4. 時間ブロックを設定

### 夕方（5分）
1. リフレクションセクションを完了
2. 未完了タスクを移動
3. `/push`を実行して変更を保存

### 週次（30分 - 日曜日）
1. ガイド付きレビューのために`/weekly`を実行
2. 目標進捗を計算
3. 来週のフォーカスを計画
4. 古いノートをアーカイブ

## ベストプラクティス

1. **具体的に** - 必要なことについて明確なコンテキストを提供
2. **目標を参照** - 日々のタスクを目標に接続
3. **コーチモードを使用** - 説明責任が必要な場合
4. **最新に保つ** - プロジェクトCLAUDE.mdファイルを定期的に更新

## カスタマイズ

コミットすべきでない個人的なオーバーライドについては、`CLAUDE.local.md`を作成してください。
形式については`CLAUDE.local.md.template`を参照してください。
