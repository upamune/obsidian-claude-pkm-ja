# 週次レビューコマンド

新しいレビューノートを作成し、先週を振り返りながら次週を計画できるよう、週次レビュープロセスを促進します。

## インストール

このファイルをvaultルートの`.claude/commands/weekly.md`にコピーします。

## 使用方法
```
claude code /weekly
```

## 設定
ワークフローに合わせてこれらの設定をカスタマイズします：

```javascript
// 設定（カスタマイズしてください）
const WEEKLY_FOLDER = "Goals";
const WEEKLY_TEMPLATE = "Templates/Weekly Review Template.md";
const REVIEW_DAY = 0; // 0=日曜日、1=月曜日、など
const TIME_INVESTMENT_TARGET = 30; // レビューの分数
```

## このコマンドが行うこと

1. **週次レビューノートを作成**
   - 週次レビューテンプレートを使用
   - 今週の日付で名前を付ける
   - Goalsフォルダに配置

2. **レビュープロセスをガイド**
   - 先週の達成事項をレビュー
   - 未完了タスクを特定
   - 来週を計画
   - 月次目標と整合

3. **ハウスキーピングを自動化**
   - 古いデイリーノートをアーカイブ
   - プロジェクトステータスを更新
   - 完了したタスクをクリーンアップ

## レビュープロセスステップ

### ステップ1: リフレクション（10分）
- 先週のデイリーノートをレビュー
- 成果と課題を特定
- 学んだ教訓をキャプチャ

### ステップ2: 目標との整合（10分）
- 月次目標の進捗をチェック
- 週次優先事項を調整
- 年間目標との整合を確保

### ステップ3: 計画（10分）
- 週の1つの大きなことを設定
- 重要なタスクをスケジュール
- 深い作業のための時間をブロック

## インタラクティブプロンプト

コマンドは以下を通じてガイドします：

1. **「今週の上位3つの成果は何でしたか？」**
   - 進捗を祝う
   - 勢いを築く
   - 達成事項を記録

2. **「主な課題は何でしたか？」**
   - 障害を特定
   - 解決策を計画
   - 困難から学ぶ

3. **「来週の1つの大きなことは何ですか？」**
   - 優先順位付けを強制
   - フォーカスを作る
   - 意味のある進捗を推進

## 週次レビューチェックリスト

コマンドは以下を支援します：
- [ ] すべてのデイリーノートをレビュー
- [ ] インボックスアイテムを処理
- [ ] プロジェクトステータスを更新
- [ ] 今後のカレンダーをチェック
- [ ] 月次目標をレビュー
- [ ] 来週の優先事項を計画
- [ ] 重要な作業のための時間をブロック
- [ ] デジタルワークスペースをクリーン
- [ ] 完了したアイテムをアーカイブ
- [ ] 変更をGitにコミット

```javascript
// 自動的に古いノートをArchives/YYYY/MM/に移動
```

## カスタマイズオプション

### 異なるレビュー曜日
```javascript
// 月曜日レビューの場合：
const REVIEW_DAY = 1;

// 金曜日レビューの場合：
const REVIEW_DAY = 5;
```

### 月次レビュー
月次レビュートリガーを追加：
```javascript
const today = new Date();
const lastDayOfMonth = new Date(today.getFullYear(), today.getMonth() + 1, 0).getDate();
if (today.getDate() === lastDayOfMonth) {
  // 月次レビューもトリガー
}
```

### スプリントベースのレビュー
アジャイルワークフロー用：
```javascript
const SPRINT_LENGTH = 14; // 日数
const SPRINT_START = new Date('2024-01-01');
// スプリント番号を計算してレビューを調整
```

## 目標との統合

コマンドは自動的に：
- [[2. Monthly Goals]]にリンク
- [[1. Yearly Goals]]の進捗を更新
- レビューログに新しい週のエントリを作成

## ベストプラクティス

### 一貫したタイミング
- 毎週同じ曜日
- 可能であれば同じ時間
- カレンダーの時間をブロック
- 譲れないものとして扱う

### 準備
- レビュー前にインボックスをクリーン
- カレンダーを用意
- プロジェクト更新を収集
- フィードバックをレビュー

### フォロースルー
- ハイライトをチーム/家族と共有
- 外部システムを更新
- 変更を伝達
- 成果を祝う

## トラブルシューティング

### レビューが作成されない？
- テンプレートが存在するか確認
- フォルダ構造を確認
- 書き込み権限を確保

### リンクが壊れている？
- ファイル命名を確認
- 日付形式をチェック
- リンク構文を更新

### 時間がかかりすぎる？
- 各セクションにタイマーを使用
- 週を通して準備
- テンプレートを簡素化
