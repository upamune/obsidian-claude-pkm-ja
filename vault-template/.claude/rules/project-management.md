# プロジェクト管理ルール

これらの規約はプロジェクトフォルダとドキュメントに適用されます。

## プロジェクト構造

各プロジェクトは独自のフォルダを持ちます：
```
Projects/
└── Project-Name/
    ├── CLAUDE.md           # 必須：AI用のプロジェクトコンテキスト
    ├── README.md           # オプション：複雑な場合の概要
    ├── notes/              # オプション：関連ノート
    └── resources/          # オプション：参考資料
```

## プロジェクトCLAUDE.md

すべてのプロジェクトには以下を含むCLAUDE.mdファイルが必要：

```markdown
# プロジェクト: [名前]

## 概要
[このプロジェクトが何であるかの1-2文の説明]

## ステータス
- **フェーズ:** 計画中 | アクティブ | 保留中 | 完了
- **進捗:** X%
- **開始:** YYYY-MM-DD
- **目標:** YYYY-MM-DD

## 目標
このプロジェクトの成功がどのように見えるか。

## 現在のフォーカス
現在積極的に取り組んでいること。

## 重要な決定
行われた重要な決定とその理由。

## 次のアクション
- [ ] 直近の次のステップ
- [ ] 次のステップ

## ブロッカー
進捗を妨げているもの。

## リソース
- [[関連ノート]]
- [外部リンク](url)

## Claudeのためのメモ
[このプロジェクトを支援する際にClaudeが知っておくべき具体的なコンテキスト]
```

## プロジェクトステータス値

一貫したステータスラベルを使用：
- `計画中` - スコープとアプローチの定義
- `アクティブ` - 現在作業中
- `保留中` - 一時的に停止（理由を記載）
- `ブロック中` - 外部依存関係を待機
- `レビュー` - 作業完了、レビュー必要
- `完了` - 完了しアーカイブ準備完了

## 進捗トラッキング

### 進捗率のガイドライン
- 0-10%：計画とセットアップ
- 10-30%：初期実装
- 30-70%：コア作業
- 70-90%：改善と仕上げ
- 90-100%：最終レビューと完了

### 更新頻度
- 最低週1回進捗を更新
- 変更時にステータスを更新
- ブロッカーは即座に更新

## プロジェクトのリンク

### 目標へ
プロジェクトを親目標にリンク：
```markdown
**サポート:** [[1. Yearly Goals#目標名]]
```

### デイリーノートへ
デイリーノートでプロジェクト作業を参照：
```markdown
## 作業ログ
- [[プロジェクト名]]に取り組んだ - Xを完了
```

### プロジェクト間
関連プロジェクトをリンク：
```markdown
**関連:** [[他のプロジェクト]]
```

## プロジェクトライフサイクル

### プロジェクトの開始
1. `Projects/`にフォルダを作成
2. テンプレートからCLAUDE.mdを作成
3. 関連する目標にリンク
4. 最初の次のアクションを定義
5. 目標完了日を設定

### プロジェクト中
1. 進捗でCLAUDE.mdを更新
2. デイリーノートに作業を記録
3. 決定と理由をキャプチャ
4. ブロッカーを即座に追跡
5. 必要に応じてタイムラインを調整

### プロジェクトの完了
1. ステータスを完了に更新
2. 完了日とサマリーを追加
3. 学んだ教訓を文書化
4. フォルダをArchives/Projects/に移動
5. 目標の進捗を更新

## ベストプラクティス

1. プロジェクトスコープを現実的に保つ
2. 一度に1つの主要フォーカス
3. 大きなプロジェクトをフェーズに分割
4. すべてのアクティブプロジェクトを週次でレビュー
5. 完了したプロジェクトを速やかにアーカイブ
6. 決定は行われたときに文書化
7. 関連するコンテキストに自由にリンク
